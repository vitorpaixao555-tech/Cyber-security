Resumo rápido — como usar esta mentoria

Objetivo: te tornar profissional de Cloud Security + Compliance aplicável em bancos. Método: cada semana tem objetivos claros, tarefas passo-a-passo, comandos/práticas, desafios e entregáveis que serão guardados neste próprio site.

  • Você cria o conteúdo do curso (labs, scripts, relatórios) enquanto aprende — tudo arquivado no site.
  • Feedback prático: peça revisão do seu projeto final; eu te oriento a ajustar para padrões Bacen.
  • Emprego: seção de carreira com roteiro de candidatura, preparação de CV/LinkedIn e simulações de entrevistas.

Plano detalhado: semana a semana (12 semanas)

Semana 1 — Preparação e ambiente

  1. Criar contas sandbox: AWS (Free Tier), Azure (trial), GCP (trial). Registrar no site as contas criadas (nome curto).
  2. Instalar ferramentas locais: AWS CLI, gcloud, az CLI, Terraform, Ansible, kubectl, Docker.
  3. Tarefa prática: criar uma VPC simples na AWS com 1 subnet pública e 1 privada. Comandos iniciais serão mostrados abaixo.
# Exemplo: criar VPC via AWS CLI
aws ec2 create-vpc --cidr-block 10.10.0.0/16 --tag-specifications 'ResourceType=vpc,Tags=[{Key=Name,Value=banco-atlas-vpc}]'

Semana 2 — IAM e políticas

  1. Entender usuários, grupos, roles e políticas gerenciadas. Aplicar princípio do menor privilégio.
  2. Tarefa prática: criar 3 roles (AdminOps limitado, AppServerRole, ReadOnlyAudit) e anexar políticas mínimas.
  3. Desafio: identificar uma política IAM vazada (simulado) e rotacionar credenciais, registrando passos no site.
# Exemplo (Terraform - role mínima)
resource "aws_iam_role" "app_role" {
  name = "atlas_app_role"
  assume_role_policy = data.aws_iam_policy_document.app_assume.json
}

Semana 3 — Criptografia e KMS

  1. Configurar chaves gerenciadas (AWS KMS) e políticas de rotação.
  2. Tarefa: criptografar um bucket S3 com KMS CMK e demonstrar acesso controlado.
# Exemplo: habilitar criptografia no bucket (Terraform)
resource "aws_s3_bucket_server_side_encryption_configuration" "example" {
  bucket = aws_s3_bucket.logs.id
  rule {
    apply_server_side_encryption_by_default {
      kms_master_key_id = aws_kms_key.atlas_key.arn
      sse_algorithm     = "aws:kms"
    }
  }
}

Semana 4 — LGPD & PCI-DSS (prático)

  1. Mapear dados sensíveis em um sistema fictício — identificar PII e dados de cartão.
  2. Tarefa: aplicar mascaramento/tokenização em uma tabela exemplo (Postgres ou DynamoDB).
  3. Entregável: matriz de tratamento de dados (where, how, retention).

Semana 5 — Logging e monitoramento

  1. Centralizar logs (CloudTrail, CloudWatch Logs / Azure Monitor / GCP Logging).
  2. Tarefa prática: configurar um pipeline básico que envia logs para S3/Cloud Storage e criar alertas para eventos de console "RootLogin" e alterações em IAM.

Semana 6 — SIEM / Playbooks de IR

  1. Montar alertas no SIEM (ex.: Elastic + Beats) e criar playbook de resposta a incidentes.
  2. Desafio: simular exfiltração de dados e executar o playbook com coleta de evidências.

Semana 7 — Proteção de dados & DLP

  1. Configurar DLP (padrões de detecção) e políticas de prevenção.
  2. Tarefa: criar uma regra que detecta exportação de arquivo com CPF/CNPJ e dispara um workflow que bloqueia e notifica segurança.

Semana 8 — Forense e análise de logs

  1. Coleta de evidências, snapshots, análise de tráfego e timeline de evento.
  2. Tarefa: realizar análise forense de um incidente simulado e documentar em formato pronto para auditoria.

Semana 9 — Infra como Código e Terraform

  1. Estruturar módulos Terraform para provisionamento seguro e estado remoto em S3 + DynamoDB (lock).
  2. Tarefa: converter recursos criados (VPC, IAM, S3) para Terraform e aplicar com validações.
# backend.tf
terraform {
  backend "s3" {
    bucket = "atlas-terraform-state"
    key    = "envs/prod/terraform.tfstate"
    region = "sa-east-1"
    dynamodb_table = "atlas-terraform-lock"
  }
}

Semana 10 — Kubernetes seguro e CI/CD

  1. Pipeline seguro: imagens assinadas, scanning (Trivy), admission controllers e RBAC.
  2. Tarefa: criar cluster EKS/GKE (sandbox) com políticas PodSecurity e pipeline que valida imagens antes do deploy.

Semana 11 — Políticas como código & OPA

  1. Escrever políticas Rego para negar recursos inseguros e bloquear configs em PRs.
  2. Tarefa: integrar OPA/Gatekeeper ao pipeline e bloquear infra com CSP insegura.

Semana 12 — Simulado final e relatório Bacen

  1. Executar um simulado completo (incidente + resposta + relatório para auditoria do Bacen).
  2. Entregável final: diagrama da arquitetura + Terraform + playbook + relatório de conformidade (evidências anexadas).

Atividades detalhadas (exemplo de um lab)

Exemplo de exercício prático — "Proteger um bucket S3 e demonstrar acesso controlado"

  1. Crie o bucket: aws s3api create-bucket --bucket atlas-logs- --region sa-east-1
  2. Habilite criptografia com KMS: criar CMK e adicionar política que só permite encr/descr para um role específico.
  3. Defina política de bucket que nega acesso público e bloqueia ACLs públicas.
  4. Teste: tente acessar o objeto sem role e depois com role; documente os logs no CloudTrail.

Estratégia de carreira — conseguir vaga em bancos

Roteiro prático, passo a passo:

  1. Currículo e GitHub: mantenha repositório com seus labs (Terraform, playbooks, relatórios). README com passo a passo e resultado.
  2. LinkedIn: resumo com palavras-chave (Cloud Security, KMS, Terraform, LGPD, PCI-DSS, Bacen). Poste resultados dos labs (cases) semanalmente.
  3. Certificações: priorize: AWS SysOps/Associate, Azure Security Engineer, e certificações de segurança como CompTIA Security+ ou (se já avançado) CISSP. Não precisa ter todas; ter 1-2 relevantes ajuda.
  4. Portfólio de entrevistas: prepare 3 estudos de caso do seu projeto final para explicar arquiteturas e trade-offs e 2 incident reports que você lidou durante o curso.
  5. Networking: participe de meetups, grupos de segurança e procure vagas de "Cloud Security Engineer" ou "Compliance Cloud" em bancos e consultorias.
  6. Simulação de entrevistas: perguntas técnicas + cenários comportamentais. Use topicos: IAM, criptografia, incident response, mapeamento de risco.
